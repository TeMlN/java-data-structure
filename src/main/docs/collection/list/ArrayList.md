### ArrayList
크기가 가변적으로 변하는 선형리스트.
순차적이며 인덱스로 내부의 객체를 관리한다

데이터의 변경이 일어나면 해당 리스트를 복사한 임의의 배열에 값을 수정하여 넣는다
즉 데이터의 변경이 일어날때마다 리스트를 복사하는 리소스가 생기게 된다 이는 성능저하의 원인이 된다.
반면 각 리스트의 원소들은 인덱스를 가지기 때문에 조회에는 용이하다.

일반적인 배열과는 달리 리스트의 길이가 정해져있지 않고 default값으로 capacity(길이)가 10으로 지정되어 있고
길이를 초과할시 더 큰 용량의 리스트를 만들어 옮기는 작업을 한다. 그렇기 때문에 ArrayList를 사용할때는 초기에 리스트의 크기를 지정해주는 것이 좋다.

#### ArrayList가 제공하는 method
* add(E element): list의 끝에 원소를 생성하여 element를 할당
* add(E element, int index): 해당 index를 가진 원소의 element값을 할당
* remove(int index): 해당 index를 가진 원소를 삭제
* get(int index): 해당 index에 해당하는 원소를 찾아옴
